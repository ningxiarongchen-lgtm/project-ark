## 📖 各角色详细工作流程

### 1️⃣ 销售经理工作流程

#### 职责定位
客户关系管理、项目全流程跟进、合同签订、款项催收

#### 4步工作流程

**第1步：创建项目并指派**
```
接收客户需求 
    ↓
系统创建项目
    ↓
上传技术文件
    ↓
指派技术工程师
```

**系统操作**：
- 点击"新建项目"
- 填写客户信息（公司、联系人、电话、地址）
- 上传客户提供的技术说明书/需求文档
- 选择技术工程师并指派
- 项目状态：`待指派技术` → `选型中`

**第2步：询价阶段**
```
等待技术选型完成
    ↓
等待商务报价完成
    ↓
下载报价单PDF
    ↓
发送给客户询价
```

**关键点**：
- ⚠️ 此时是**询价阶段**，不是赢单
- 项目状态：`已报价-询价中`
- 客户可能接受或拒绝

**第3步：赢单阶段（签约）**
```
客户接受报价
    ↓
上传销售合同
    ↓
等待商务审核盖章
    ↓
下载盖章合同给客户
    ↓
客户签字盖章
    ↓
上传最终合同
    ↓
正式赢单 ✅
```

**状态流转**：
- `已报价-询价中` → `待上传合同`
- `待上传合同` → `待商务审核合同`
- `待商务审核合同` → `待客户盖章`
- `待客户盖章` → `合同已签订-赢单` ✅

**第4步：跟进生产与催款**
```
配合商务催收预付款
    ↓
预付款到账
    ↓
项目进入生产阶段
    ↓
跟踪生产进度
    ↓
发货后催收尾款
    ↓
尾款到账，项目完成
```

#### 核心指标
- 📊 月度新增项目数
- 💰 月度赢单金额
- ⏱️ 平均报价响应时间（目标：<24小时）
- 📈 赢单率（目标：>30%）
- 😊 客户满意度（目标：>90%）

#### 职责定位
技术选型与配置、售后技术支持

#### 双重职责

**职责A：技术选型（70%工作量）**

```
接收销售指派的项目
    ↓
下载客户技术文件
    ↓
分析技术要求
    ↓
使用智能选型系统
    ↓
选择执行器型号
    ↓
配置附件和配件
    ↓
填写技术BOM清单
    ↓
提交给商务报价
```

**智能选型工具**：
- 单个选型：逐个处理项目
- 批量选型：一次处理多个项目（推荐）
- 输入参数：扭矩、压力、阀门类型、温度、旋转角度等
- 系统自动推荐最优产品方案

**职责B：售后服务（30%工作量）**

```
接收售后工单
    ↓
查看问题描述
    ↓
接受任务
    ↓
诊断问题原因
    ↓
提供解决方案
    ↓
远程指导/现场处理
    ↓
编写解决报告
    ↓
标记已解决
    ↓
等待客户确认
    ↓
工单关闭
```

**工单类型**：
- 🔧 维修：设备故障、零件损坏
- 📦 安装：现场安装指导
- 🔍 检查：定期检查、故障诊断
- 📚 培训：操作培训、技术指导
- 💬 咨询：技术问题咨询
- ⚠️ 投诉：客户投诉处理

#### 核心指标
- ⚡ 选型准确率（目标：>95%）
- ⏱️ 平均选型时间（目标：<2小时）
- 🛠️ 售后响应时间：紧急<4小时，高优先级<24小时
- 😊 客户满意度（目标：>90%）

#### 职责定位

```

    ↓
审核合同内容
├─ 客户信息
├─ 金额与报价单对比
├─ 付款条款
├─ 交付日期
├─ 法律条款
└─ 决策
    ├─ ✅ 无问题 → 盖章流程
    └─ ❌ 有问题 → 驳回修改
        ↓
线下盖章流程
├─ 打印合同
├─ 领导审批
├─ 加盖公章
└─ 扫描PDF
    ↓
上传盖章版合同
    ↓
通知销售经理
    ↓
销售转交客户签字
```

#### 工作流程B：采购合同

```
接收采购合同
    ↓
审核合同内容
├─ 供应商资质
├─ 采购清单
├─ 价格与预算
├─ 交货期
├─ 质量标准
└─ 决策
    ├─ ✅ 无问题 → 盖章流程
    └─ ❌ 有问题 → 驳回修改
        ↓
线下盖章流程
├─ 打印合同
├─ 采购审批
├─ 加盖采购章
└─ 扫描PDF
    ↓
上传盖章版合同
    ↓
通知采购专员
    ↓
采购发给供应商签字
```

#### 合同管理中心功能

**统计看板**：
- 📊 合同总数
- 🔔 待盖章数量（需要处理）
- ✅ 已盖章数量
- ⏰ 即将到期提醒
- 💰 待催款合同

**快速操作**：
- 📥 下载草稿合同
- ✅ 接单处理
- 📤 上传盖章版
- ❌ 驳回合同
- 📝 添加跟进记录

#### 核心指标
- ⏱️ 审核及时性（目标：<2个工作日）
- ✅ 审核准确性（错误率<1%）
- 📊 月度处理合同数
- 🎯 一次通过率（目标：>80%）

#### 物料齐套管理

**齐套状态**：
- `待分析` - 尚未检查
- `部分可用` - 部分物料有库存
- `全部可用(齐套)` - 所有物料齐套，可以生产 ✅
- `采购延迟` - 采购未到货

**缺料处理**：
```javascript
{
  item_name: "电动执行器 AT-GY-100",
  required_quantity: 10,      // 需求数量
  available_quantity: 5,      // 可用库存
  shortage_quantity: 5,       // 缺料数量
  purchase_order_id: "PO-001",
  expected_arrival_date: "2025-11-05"
}
```

#### 核心指标
- ⏱️ 准时交付率（目标：>90%）
- 📊 生产周期（目标：比计划缩短20%）
- 🎯 物料齐套及时率（目标：>85%）
- 📈 产能利用率（目标：>80%）

## 附录：快速问答

**Q1: 系统真的几乎免费吗？**  
A: 是的。使用Vercel、Render、MongoDB Atlas的免费版本，年成本不到100元（域名费用）。

**Q2: 系统稳定吗？**  
A: 非常稳定。基于成熟的云服务，99.9%可用性保证，自动备份，无需担心数据丢失。

**Q3: 上线需要多长时间？**  
A: 推荐方案：1周。包括部署、数据导入、培训、试运行。

**Q4: 员工能学会吗？**  
A: 可以。界面友好，操作简单，提供完整培训。每个角色都有专属的Dashboard和工作流程指引。

**Q5: 如果不满意怎么办？**  
A: 几乎零成本，可随时停用。但根据测试，满意度预期达到90%以上。

**Q6: 可以定制开发吗？**  
A: 可以。系统具有高度可扩展性，可根据实际需求快速迭代。

**Q7: 数据安全吗？**  
A: 安全。使用JWT认证、HTTPS加密传输、权限严格控制、自动备份。

**Q8: 需要招聘专门的IT人员吗？**  
A: 不需要。日常维护由现有员工兼职即可，遇到问题有技术团队支持。

**Q9: 各部门之间如何协同？**  
A: 系统自动流转。例如：销售创建项目→自动通知技术选型→自动通知商务报价。每个环节完成后，系统自动通知下一步负责人。

**Q10: 角色权限如何控制？**  
A: 每个角色只能看到和操作自己职责范围的内容。例如：车间工人只能看到分配给自己的工单，技术工程师看不到价格信息。
